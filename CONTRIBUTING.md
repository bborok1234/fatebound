# 기여 가이드 · Contributing

Fatebound(천명회귀)에 관심 가져 주셔서 고맙습니다. 1인 + AI 운영 프로젝트라 기여 흐름은 가볍게 유지합니다.

## 시작하기

```bash
git clone https://github.com/bborok1234/fatebound.git
cd fatebound/game
uv sync                 # 의존성 + dev 도구 설치 (uv.lock 기반 재현)
uv run fatebound        # 게임 실행
uv run pytest -q        # 테스트 (통과해야 PR 머지)
uv run fatebound --selftest   # 비대화 자가검증
```

[uv](https://docs.astral.sh/uv/)를 패키지/환경 관리에 사용합니다. `pip install -e .`로도 가능하지만 기여는 uv 흐름을 권장합니다.

## 무엇을 기여할 수 있나

- 🐛 **버그 수정** — 재현 절차를 이슈에 적어 주세요.
- 🎨 **TUI/UX 개선** — 위젯·테마·접근성·터미널 호환성.
- ⚙️ **엔진 개선** — `engine/`은 I/O 없는 순수 로직입니다. 결정론(시드 동일 → 결과 동일)을 깨지 마세요.
- 🌍 **터미널/플랫폼 호환성** — 다양한 터미널·OS에서의 렌더링 제보.

> **참고:** 게임의 상세 기획(세계관·밸런스 수치 설계·콘텐츠 로드맵)은 비공개로 관리됩니다. 따라서 대규모 콘텐츠/밸런스 방향 제안은 PR보다 **이슈로 먼저 논의**해 주세요. 메인테이너가 비공개 설계와 대조해 답을 드립니다.

## 코드 규칙

- **언어/스타일**: Python 3.11+. 표준 라이브러리 우선, 의존성 추가는 이슈에서 합의 후.
- **`engine/`는 순수하게**: 파일 I/O·네트워크·`print`를 두지 마세요. 출력은 이벤트(`engine.events`)로 emit하고 표현층(TUI/텍스트)이 처리합니다.
- **밸런스 상수**는 `engine/balance.py` 한 곳(SSOT)에만. 매직 넘버를 흩뿌리지 마세요.
- **결정론**: 무작위는 반드시 시드 가능한 `engine.rng.Rng`로. `random`/`time` 직접 호출 금지.
- **테스트**: 동작을 바꾸면 `tests/`에 회귀 테스트를 추가하세요. 특히 빌드별 보스 공략 가능성(죽은 빌드 0) 가드를 깨지 마세요.

## PR 절차

1. 포크 → 브랜치(`fix/...`, `feat/...`).
2. `uv run pytest -q` 그린 확인.
3. PR 템플릿을 채우고 변경 이유를 설명.
4. CI(리눅스·맥·윈도우 × Python 3.11~3.13)가 그린이어야 머지됩니다.
